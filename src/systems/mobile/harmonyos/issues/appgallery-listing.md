# AppGallery 上架流程（知识沉淀）

> 本文件是 ForgeKit 在鸿蒙打包完成后向用户提示"下一步该怎么做"的依据。
> 用户无需记住流程，ForgeKit 会按当前产物状态给出对应下一步。

## 完整上架路径
1. **实名认证**：注册华为开发者账号，完成个人或企业实名认证。
2. **创建应用**：在 AppGallery Connect 创建应用，填写 `bundleName`（须与工程一致）与应用分类。
3. **申请证书**：
   - 本地用 `keytool` 或 DevEco 生成密钥库 `.p12` 与证书请求 `.csr`。
   - 上传 `.csr` 到 AGC → 签发正式证书 `.cer`。
4. **创建 Profile**：在 AGC 创建发布 Profile `.p7b`，绑定证书与应用。
5. **配置签名**：将 `.cer`/`.p12`/`.p7b` 配置进 `build-profile.json5` 的 `signingConfigs`。
6. **构建发布包**：`hvigorw assembleApp`（release），得到 `*.app`。
7. **提交审核**：AGC 上传 `.app`，填写应用信息、截图、隐私政策、权限说明，提交人工审核。
8. **发布**：审核通过后发布或分阶段发布。

## 状态机（ForgeKit 用于提示）
- 未实名 → 提示"先完成华为开发者实名认证"
- 未创建应用 → 提示"在 AGC 创建应用并核对 bundleName"
- 未签发证书/Profile → 提示"生成 CSR 并签发正式证书与 Profile"
- 未正式签名 → 提示"配置 release signingConfigs"
- 已构建 release APP → 提示"上传 AGC 提交审核"
- 审核驳回 → 提示"按驳回意见修正后重新提交"

## 合规清单（提交前自检）
- [ ] 隐私政策链接可公开访问
- [ ] 所有申请的权限均有用途说明
- [ ] 无受限权限的越权使用
- [ ] 完成内容分级
- [ ] 包体为正式签名（非 debug）
- [ ] 截图/描述与实际功能一致
