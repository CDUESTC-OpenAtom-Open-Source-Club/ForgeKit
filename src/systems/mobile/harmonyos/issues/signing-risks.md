# 鸿蒙签名风险与最佳实践

> 鸿蒙签名是上架的"命门"，ForgeKit 在 `pack_harmonyos_app` 合规预检中会重点提示以下风险。

## 1. 证书/密钥丢失
- **影响**：`.p12`（密钥库）丢失后，无法再对**已上架应用**发布更新版本（华为要求同签名续版）。
- **建议**：离线加密备份 `.p12` 与密码；纳入团队密钥管理，不进 Git、不进镜像。

## 2. 调试证书不可上架
- **影响**：使用 DevEco 自动生成的 debug 证书/Profile 打出的 APP，审核直接驳回。
- **建议**：上架前切换到 AGC 签发的 release 签名（正式 `.cer` + `.p7b`）。

## 3. Profile 绑定关系
- `.p7b` Profile 与 `bundleName`、证书、设备绑定。
- 更换 bundleName、证书或新增上架设备时，需**重新生成 Profile** 并重新构建。

## 4. 权限与合规
- 受限权限需在 AGC 申请并填写用途说明。
- 隐私政策必须真实可访问，否则审核驳回。

## 5. 安全红线
- 不要把 `.p12` 密码、`.cer` 写入 `Dockerfile`、`build-profile.json5` 明文（可用环境变量占位或在 CI 注入）。
- 不要把私钥/证书提交到公开仓库。
