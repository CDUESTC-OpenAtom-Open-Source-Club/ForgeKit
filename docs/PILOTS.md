## 1. 已拍板决策（v0.1）

| 决策项 | 结论 | 依据 |
|--------|------|------|
| **Docker-first 策略** | ✅ 已拍板："Docker 硬闭环 + deb 可选" | Docker 降低第一步门槛，deb 仅在 Ubuntu + systemd 场景提供 |
| **MVP 范围收窄** | ✅ 已拍板：Python + Docker（硬闭环） + deb（可选） | 先验证单语言闭环，v0.2 扩展多语言 |

## 4. 试点项目要求

| 要求 | 说明 |
|------|------|
| **真实项目** | 必须是真实可运行的项目，不是虚构或测试项目 |
| **有真实痛点** | 当前打包/发布流程存在困难或复杂度 |
| **愿意参与验证** | 项目负责人愿意在 v0.1 阶段参与试点验证 |
| **时间窗口** | 能在 v0.1 开发周期（2-3周）内完成验证 |
| **反馈意愿** | 愿意提供使用反馈、问题报告、改进建议 |

---

## 5. 试点验证流程

1. **试点准备**：填写试点模板，确认项目信息。
2. **v0.1 开发**：ForgeKit 实现 Docker 硬闭环能力。
3. **试点执行**：用试点项目实际验证 ForgeKit 打包流程。
4. **问题收集**：记录问题、错误、改进建议。
5. **反馈迭代**：根据反馈优化 ForgeKit。
6. **验收通过**：试点项目成功生成可运行产物。

---

## 6. 试点成功标准

| 标准 | 验收方式 |
|------|----------|
| Docker 镜像可运行 | `docker run <image>` 成功启动并响应 |
| 构建过程可解释 | Forge.md 包含清晰的决策依据 |
| 失败有诊断 | 构建失败返回可读错误摘要和修复建议 |
| 用户可复现 | 用户可在文档指导下独立完成一次打包 |
