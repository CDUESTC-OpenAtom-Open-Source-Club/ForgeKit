## 一、v0.1架构改进总结

### 1.1 已解决的架构问题

#### ✅ 问题1：TypeScript编译环境不一致

**问题现象**：
- 本地编译产物使用相对路径（正确）
- 服务器编译产物使用path aliases（Node.js无法解析）

**根本原因**：
- tsconfig.json配置了path aliases但源代码未使用
- TypeScript编译器在不同环境下处理paths行为不一致

**解决方案**：
- 移除tsconfig.json中的baseUrl和paths配置
- 源代码保持使用相对路径
- 确保编译产物在不同环境下一致

**验证结果**：
```bash
# 编译前后对比
Before: require("@capabilities/inspect-project.js")  # ❌ Node.js无法解析
After:  require("../../capabilities/inspect-project.js")  # ✅ 可移植

# 测试结果
npm test: 72/72 passed ✅
```

#### ✅ 问题2：ESLint配置缺失

**问题**：测试文件需要独立的tsconfig配置

**解决方案**：
- 创建tsconfig.eslint.json包含测试文件
- 更新.eslintrc.js指向新配置

**效果**：新环境部署时ESLint可正常工作

#### ✅ 问题3：项目结构规范化

**改进内容**：
- 移动示例文件Forge.md到docs/specs/
- 删除临时文件.gitignore.new
- 归档过期文档docs/beian-page/到archive/

**效果**：项目根目录整洁，文档结构清晰

**目标**：构建失败时提供可读诊断+修复建议

**架构设计**：
```typescript

export interface ErrorDiagnostic {
  code: string;              // 错误码
  summary: string;           // 简短描述
  detail?: string;           // 详细日志
  suggested_fix: string;     // 修复建议
  related_rules?: string[];  // 相关决策规则
}

export class ErrorDiagnostician {
  private rules: Map<string, DiagnosticRule>;

  diagnose(error: Error, context: BuildContext): ErrorDiagnostic {
    // 1. 匹配已知错误模式
    const matchedRule = this.matchPattern(error);

    // 2. 生成诊断结果
    return {
      code: matchedRule.code,
      summary: matchedRule.summary,
      suggested_fix: this.generateFix(matchedRule, context)
    };

    // - 权限问题："permission denied"
    // - Docker守护进程："docker daemon not running"
```

```yaml
version: "1.0"
platform: ubuntu

base_images:
  - distribution: ubuntu-22.04
    base_image: python:3.10-slim
    glibc_version: "2.35"
    python_versions: ["3.8", "3.9", "3.10", "3.11", "3.12"]

  version_selection:
    - condition: "target_distribution == 'ubuntu-22.04'"
      recommendation: "选择 Python 3.10 LTS"
      reason: "平衡稳定性和新特性"

    - condition: "target_architecture == 'arm64'"
      recommendation: "使用 python:3.10-slim-arm64v8"

      diagnosis:

        summary: "权限不足"
        suggested_fix: "检查文件权限，或使用sudo运行"
```
