# v0.2 迭代计划（轻量级骨架）

> **核心目标**：从能跑到能用 - 错误诊断 + 真实试点 + 扩展能力

## 范围控制原则

✅ **保持轻量**：
- 只做必要功能，不做大重构
- 优先解决真实用户痛点
- 代码量控制在现有基础上增加 < 30%

❌ **明确不做**：
- 不做移动端（v0.3）
- 不做云构建（L2层）
- 不做复杂UI（纯MCP工具）

---

## v0.2 三个核心任务

### 任务1：错误诊断增强（1-2周）

**目标**：构建失败时提供可读诊断 + 修复建议

**范围**：
```typescript
// 新增：错误诊断器
src/capabilities/utils/error-diagnostic.ts

功能：
- Docker 构建失败 → 解析日志 → 定位原因
- 依赖缺失 → 推荐安装命令
- 配置错误 → 提供修复模板
```

**验收**：
- 常见错误（依赖缺失、端口冲突、权限问题）有明确诊断
- 用户无需翻完整日志即可理解失败原因

---

### 任务2：真实项目试点（并行进行）

**目标**：征集 3 个真实项目验证闭环

**范围**：
```
试点候选：
1. ✅ sample-python-project（已有）
2. 🎯 TypeScript Node.js 项目（新增 fixture）
3. 🎯 Go 项目（新增 fixture）

验证重点：
- Agent 调用成功率
- 错误可解释性
- 用户反馈收集
```

**验收**：
- 3 个不同语言项目完成 Docker 构建
- 收集至少 5 条真实用户反馈

---

### 任务3：扩展支持（可选，按需）

**目标**：补齐服务器端基础能力

**范围**：
```
优先级 P0（必做）：
- ✅ 多架构支持（linux/amd64 + linux/arm64）
- ✅ TypeScript 项目模板

优先级 P1（可选）：
- 🔘 rpm 支持（仅当有真实需求）
- 🔘 更多 Python 版本（3.8-3.12）
```

**验收**：
- ARM64 镜像构建成功
- TypeScript 项目模板可用
