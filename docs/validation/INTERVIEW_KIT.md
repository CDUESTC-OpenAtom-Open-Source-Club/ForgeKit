# 访谈执行包

## 中立邀请语

你好，我在参与一个社团开源项目的需求验证。看到你在公开 Issue 中讨论了 Docker 构建/部署问题。我们想了解你目前如何解决这类问题，以及现有工具是否已经够用。

这不是销售访谈，也不预设你需要新工具。访谈约 20–30 分钟；如果你愿意，也可以只用文字回答。我们不会公开私人信息，公开结果只做匿名汇总。

## 核心问题

1. 最近一次相关构建或部署任务是什么？
2. 当时具体哪里失败，最终怎样解决，花了多久？
3. 你现在使用 Docker、CI、静态检查或 Agent 的方式是什么？
4. 现有方案哪里已经足够好？哪里仍需要人工判断？
5. 如果只能选一项，你更需要“Agent 自动打包”还是“解释并验证 Docker 构建”？为什么？
6. 你愿意拿一个公开或脱敏项目完成 15 分钟试用吗？
7. 试用后的一周内，你是否愿意再次使用？

避免直接问“如果有这个功能你会不会用”。必须先记录过去行为和替代方案，再展示 ForgeKit。

## 记录字段

| 字段 | 内容 |
|---|---|
| 用户编号 | 匿名编号 |
| 项目类型 | Agent/MCP/普通服务 |
| 最近任务 | 用户实际完成过的任务 |
| 当前方案 | Docker/CI/平台/人工流程 |
| 失败与耗时 | 原话和可验证事实 |
| 偏好 | 打包技能 / 构建诊断 / 都不需要 |
| 愿意试用 | 是 / 否 / 需考虑 |
| 完成试用 | 是 / 否 |
| 第二次使用 | 是 / 否 |
| 反对意见 | 现有方案足够、集成成本等 |
