## 发布动作

外部硬门禁完成后，按以下顺序执行：

```bash
git tag v0.2.2-rc.1
git push origin v0.2.2-rc.1
npm publish --tag next
gh release create v0.2.2-rc.1 --prerelease --generate-notes
```

受控 GitHub Actions 入口见 `.github/workflows/release-rc.yml`。它要求传入 Linux 与 HarmonyOS 证据引用，执行 `npm run verify` 和 `npm run gate:diagnostics`，并只使用 npm `next` dist-tag。严重问题时停止 `next` 推广，保留日志和复现命令，不执行破坏性 unpublish。

## 未纳入承诺

Android、iOS、PWA、Windows、rpm 不属于本 RC 的发布承诺。HarmonyOS 只有在真实 DevEco 构建和设备安装证据归档后，才能从 `experimental` 变为 `verified`。
